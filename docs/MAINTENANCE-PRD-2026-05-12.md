## 1. 产品目标

### 1.1 核心目标

为 DMS 系统构建设备维护周期提醒功能，帮助制药企业实现：

- 设备预防性维护的周期化管理
- 维护到期自动预警
- 维护记录可追溯，满足 GMP 合规要求

### 1.2 产品定位

作为 DMS 系统"生命周期管理"模块的增量功能，补充现有"校准计划管理"之外的维护提醒能力。

## 2. 用户故事

### US-MNT-01：设置维护周期

**作为** 设备工程师
**我希望** 能为每台设备设置维护周期
**以便** 系统能自动提醒我何时需要进行维护

**验收标准**：
- 可选择固定周期（7天/30天/90天/180天/365天）
- 可自定义周期天数（1-365天）
- 支持为同一设备设置多种维护类型（校准、保养、巡检）

---

### US-MNT-02：查看到期提醒

**作为** 设备工程师
**我希望** 登录系统后能看到即将到期的维护设备
**以便** 我能及时安排维护工作

**验收标准**：
- 登录时弹窗显示7天内即将到期的设备
- 首页看板显示到期设备列表
- 支持按维护类型筛选

---

### US-MNT-03：记录维护结果

**作为** 设备工程师
**我希望** 能在完成维护后记录维护内容和结果
**以便** 形成完整的维护历史记录

**验收标准**：
- 可填写维护内容描述
- 可选择维护结果（合格/不合格/待处理）
- 维护完成后自动计算并更新下次维护时间

---

### US-MNT-04：查看维护历史

**作为** QA经理
**我希望** 能查看设备的维护历史记录
**以便** 在审计时提供合规证据

**验收标准**：
- 设备详情页展示该设备的所有维护记录
- 维护记录包含：维护时间、维护类型、维护内容、维护结果、执行人

## 3. 需求池

### 3.1 功能需求

| ID | 功能点 | 描述 | 优先级 | 工作量 |
|----|--------|------|--------|--------|
| F-01 | 维护周期设置 | 为设备设置/编辑维护周期（类型、周期天数） | P0 | M |
| F-02 | 到期设备看板 | 首页展示到期设备列表 | P0 | L |
| F-03 | 登录弹窗提醒 | 用户登录时弹出即将到期设备 | P0 | L |
| F-04 | 维护记录登记 | 记录维护内容、结果、更新时间 | P0 | M |

## 4. 字段设计

### 4.1 维护计划表 (maintenance_plan)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| device_id | UUID | 是 | 关联设备ID |
| maintenance_type | ENUM | 是 | 维护类型：校准/保养/巡检 |
| interval_days | INTEGER | 是 | 周期天数 |
| next_due_date | DATE | 是 | 下次到期日 |
| is_active | BOOLEAN | 是 | 是否启用 |
| created_by | UUID | 是 | 创建人 |
| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |

### 4.2 维护记录表 (maintenance_record)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| plan_id | UUID | 是 | 关联计划ID |
| device_id | UUID | 是 | 关联设备ID |
| maintenance_type | ENUM | 是 | 维护类型 |
| content | TEXT | 是 | 维护内容 |
| result | ENUM | 是 | 维护结果：合格/不合格/待处理 |
| performed_by | UUID | 是 | 执行人 |
| performed_at | TIMESTAMP | 是 | 维护时间 |
| next_due_date | DATE | 是 | 本次记录后计算的下次到期日 |
| created_at | TIMESTAMP | 是 | 记录创建时间 |

## 6. 业务规则

### 6.1 周期计算规则

- **到期日计算**：`上次维护日期 + 周期天数`
- **首次设置**：默认从当前日期开始计算
- **提前提醒**：
  - 7天内到期：高亮显示
  - 今日到期：红色警告
  - 已过期：红色标记，计入逾期统计

### 6.2 维护记录触发规则

- 提交维护记录后，自动更新对应计划的下次到期日
- 不合格/待处理状态：保持原到期日不变，不更新周期
- 维护记录记入审计日志

### 6.3 提醒推送规则

- 登录提醒：仅对有到期设备的用户显示
- 看板数据：实时更新，支持按部门/设备类别筛选
- 提醒关闭：用户可选择"7天内不再提醒"

## 8. 技术实现要点

### 8.1 数据库表

参考字段设计章节 (4.1, 4.2)

### 8.2 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/devices/{id}/maintenance-plans` | GET | 获取设备维护计划 |
| `/api/devices/{id}/maintenance-plans` | POST | 创建设备维护计划 |
| `/api/devices/{id}/maintenance-plans/{plan_id}` | PUT | 更新维护计划 |
| `/api/devices/{id}/maintenance-plans/{plan_id}` | DELETE | 删除维护计划 |
| `/api/maintenance-records` | POST | 提交维护记录 |
| `/api/maintenance-records` | GET | 查询维护记录（支持设备ID筛选） |
| `/api/dashboard/due-maintenance` | GET | 获取到期设备看板数据 |

### 8.3 登录提醒实现

- 用户登录成功后，异步查询当前用户的关联设备中是否有7日内到期项
- 如有，渲染提醒弹窗组件
- 弹窗关闭状态存储于 localStorage，7天内不重复显示

## 9. 依赖与风险

### 9.1 功能依赖

| 依赖项 | 说明 |
|--------|------|
| 设备管理模块 | 需先完成设备 CRUD 功能 |

### 9.2 风险项

| 风险 | 影响 | 缓解措施 |
